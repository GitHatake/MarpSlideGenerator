# Marp Slide Generator

プロジェクトの成果物や設計資料から、Marpを活用して視認性が高く美しいプレゼンテーションスライドを自動生成するためのツールです。親プロジェクトにサブモジュールとして組み込んで使用することを前提に設計されています。

## 🚀 親プロジェクトへの導入手順 (サブモジュール)

あなたのプロジェクトにこのスライド生成ツールを導入する手順です。

1. **サブモジュールの追加**
   プロジェクトのルートディレクトリで、このリポジトリをサブモジュールとして追加します。
   ```bash
   git submodule add git@github.com:GitHatake/MarpSlideGenerator.git slide-generator
   ```

2. **依存関係のインストール**
   追加されたディレクトリに移動し、必要なパッケージをインストールします。
   ```bash
   cd slide-generator
   npm install
   ```

3. **コミット**
   親プロジェクトに戻り、サブモジュールの追加をコミットします。
   ```bash
   cd ..
   git add .gitmodules slide-generator
   git commit -m "Add slide-generator submodule"
   ```

## 🌿 プロジェクト（ブランチ）ごとのスライド環境のセットアップ

1つのリポジトリで複数のプロジェクトやバージョンのスライドを管理するために、ブランチごとに専用の出力ディレクトリを作成するワークフローを採用しています。

新しくスライドを作成する際は、以下のスクリプトを実行して専用のブランチとディレクトリをセットアップしてください。

```bash
cd slide-generator
./branch_setup.sh <新しいブランチ名>
```

これにより、以下の設定が自動的に行われます。
1. `<新しいブランチ名>` というブランチが作成されます。
2. `slides/<新しいブランチ名>/` という専用ディレクトリが作成されます。
3. エージェント用のプロンプトが専用ディレクトリにコピーされます。
4. `package.json` の生成先パスが、この専用ディレクトリを向くように書き換えられます。

## 🤖 エージェント（Gemini等）を用いたスライド作成の自動化ワークフロー

専用ブランチのセットアップが完了したら、エージェントに対して以下のようなプロンプトを与えてください。

### エージェントへの指示方法（プロンプト例）

```text
あなたはこのプロジェクトの資料をもとに、プレゼンテーションスライドを作成するエージェントです。

以下の手順で作業を行ってください。

1. 【ルールの確認】
   現在作業中のブランチ名（例: `feature-x-pitch`）を確認し、`slide-generator/slides/<ブランチ名>/system_prompt.md` の内容を読み込んでください。
   そこに記述されているスライドデータのスキーマ定義と生成ルール（特に出力すべき JSON フォーマットの制約）を完全に理解し、厳守してください。

2. 【情報の抽出とスライド構成の作成】
   プロジェクト内の以下のファイル（※スライド化したいファイルパスを指定）を読み込み、内容を分析してください。
   ・ `docs/requirements.md`
   ・ `README.md`
   
   分析した内容をもとに、上記で確認したルールに従って `slideData` （JSON配列）を生成してください。

3. 【ファイルの保存】
   生成したJSONデータを、`slide-generator/slides/<ブランチ名>/slide_input.json` というファイル名で保存してください。
   ※必ずプレーンなJSON形式で保存し、Markdownのコードブロック記法（```json など）は含めないでください。

4. 【スライドのビルド】
   保存が完了したら、`slide-generator` ディレクトリに移動し、以下のコマンドを実行してスライド（PDFおよびMarkdown）を生成してください。
   ```bash
   cd slide-generator && npm run generate
   ```

5. 【完了の報告】
   ビルドが成功したら、生成されたファイル（`slide-generator/slides/<ブランチ名>/output.pdf` など）の場所を私に報告してください。
```

## 🛠️ 手動での生成コマンド

エージェントを使わず手動で生成する場合は、以下の手順で行います。

1. `./branch_setup.sh <ブランチ名>` を実行済みであることを確認します。
2. `slide-generator/slides/<ブランチ名>/slide_input.json` にJSONデータを配置します。
3. `slide-generator` ディレクトリ内で以下のコマンドを実行します。

```bash
# MarkdownとPDFを一括生成 (専用ディレクトリに出力されます)
npm run generate

# プレビューしながら調整したい場合
npm run preview

# HTML形式で出力したい場合
npm run build:html

# PowerPoint形式(pptx)で出力したい場合
npm run build:pptx
```

## 🎨 スタイルのカスタマイズ
`themes/custom-theme.css` 内の `:root` セクションにあるカラーコード（`--color-primary` など）を変更するだけで、プロジェクトのブランドカラーに合わせたスライドデザインに即座に変更できます。
